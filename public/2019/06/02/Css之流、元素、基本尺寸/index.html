CSS之流、元素、基本尺寸

2019-06-02 • 标签: css

css的历史环境

css的全称是 Cascading Style Sheets，翻译成中文是 层叠样式表。所谓 层叠，意思是样式可以层层叠加，比如说，一个页面的元素都继承了12像素的大小，某标题就可以设置成14像素进行叠加，后面写的样式可以覆盖先前写的同名样式，相当灵活。
css1 1996.12.17
css2 1998.5.12 推行内容和表现分离，table布局开始落寞
css3 2001.5.23 主要包括盒子模型、列表模块、超链接方式、语言模块、背景和边框、文字特效、多栏布局等模块
css世界的诞生就是为图文信息展示服务的。

文档流

流 是CSS世界中的一种基本的定位和布局机制，可以理解为现实时间的一套物理规则， 流 和现实世界的 水流 有异曲同工的表现。
eg:
拿 div 和 span 举例，分别是 块级元素 和 内联元素，分别对应容器里的水和木头，水是自动铺满容器的，木头依次排列。
所以， 流 就是 CSS 世界中 引导元素排列和定位 的一条看不见的 水流。

流是如何影响整个CSS世界的（CSS是如何设计的？）

整个CSS世界几乎就是围绕流来建立的。
那么流是如何影响整个CSS世界的？

  1. 让 HTML 默认的表现符合 流，（CSS是继承层叠的）
  2. 实现复杂的布局，就要打破坏 流
  3. 改变流向，默认的流向是从左往右，自上而下

流体布局

指的是利用元素 流 的特性实现的各类布局效果。 流 本身都具有自适应性， 所以流体布局是也是有自适应性的。但是 流体布局 ≠ 自适应布局。自适应布局是对具有自适应特性的一类布局的统称。
所以，自适应布局 > 流体布局。

未定义行为

在我们日常排版中，会遇到某些css在不同浏览器表现形式不一致的情况，会认为是某个浏览器的bug。
需要转变一下想法，web标准不会面面俱到的，规范描述未触达的地方，需要各个浏览器厂商自己去想象。
比如 FireFox mousedown干掉:active实例页面
FireFox认为:active发生在mousedown事件之后

流、元素与基本尺寸

HTML通常分为两类：块级元素和内联元素

块级元素 （block-level element）

基本特征： 一个水平流上只能单独显示一个元素（也不用占满，比如 <table> 的宽度是由内容决定大小的，就算只有一列，也自己独占一行），多个块级元素则换行显示。
常见的块级元素有 <div> 、 <li> 、 <table> ，注意 块级元素 和 display: block 不是一个概念。
比如 <li> 元素的display的默认值是list-item， <table> 元素的display的默认值是table，但是他们都叫 块级元素 ，因为他们都符合块级元素的基本特征。
作用：
由于 块级元素 具有换行特性，所以理论上它都可以配合 clear 属性来清除浮动带来的影响
clear属性是float的克星，官方属性解释： 元素盒子的边不能和前面的浮动元素相邻。其值为 none | left | right | both
clear属性只有块级元素才有效的，而我们平常都会写

但是在实际开发中，我们不会用list-item来清除浮动

  1. 字符比较多
  2. 会出现不需要的项目符号，需要加list-style: none来去除
  3. IE不支持伪元素的display的值为list-item

盒子的类型

盒子的类型种类是不断迭代的。

  1. 块级盒子

    负责结构
  2. 内联盒子

    负责内容，不管结构，比如 <span> 直接设置 height或者width是没有用的
  3. 附加盒子（标记盒子marker box）

    比如，因为引入 <ul>、<ol>、<li>标签，需要显示项目符号，所以新增了一个新的 display 属性值 list-item，

    <li> 标签的默认的 display 属性值就是 list-item。

    所以在之前两个盒子的基础之上，所有 块级元素 都有一个 主块级盒子，而list-item除此之外还有一个 附加盒子。
  4. 外在盒子和内在盒子

    display的值又出现了 inline-block，它的表现是穿着inline的皮藏着block的心，之前定义的盒子没法儿解释这个现象，

    然后导致每个元素都用两个盒子表示：外在盒子和内在盒子 。

    外在盒子负责元素是可以一行显示，还是只能换行显示（即体现元素是内联元素还是块级元素）；内在盒子负责宽高、内容呈现等。（内在盒子又称为“容器盒子”）

    对标兼容解释：

    display: block 的元素的盒子由外在的“块级盒子”和内在的“块级容器盒子”组成，

    display: inline-block 的元素则由外在的“内联盒子”和内在的“块级容器盒子”组成，

    display: inline 的元素则内外均是“内联盒子”。

    所以display属性值是inline-block的元素既能和图文一行显示，又能直接设置width/height （作用在内在盒子上），

    因为有两个盒子，外面的盒子是inline级别，里面的盒子是block级别。

    测试：

    display: inline-table 的表现形式

    外面是 内联盒子 ，里面是 table盒子，得到一个可以和文字在一行显示的表格。

width/height作用在哪个盒子上

内在盒子上

width:auto;

css中，width的默认值是auto。但是它的表现形式有4种。

充分利用可用空间

width: -webkit-fill-available;
比如 <div>、<p>这种块级元素的宽度默认是 100%于父级容器的

收缩与包裹

width: -webkit-fit-content;
收缩到合适，比如浮动，绝对定位，inline-block元素或者table元素

收缩到最小

width: -webkit-min-content;
这个最容易出现在table-layout为automatic的表格中
当每一列空间都不够的时候，文字能断就断，但是中文是可以随便断的，英文单词不能断

超出容器限制

width: -webkit-max-content;
比如内容很长的连续的英文和数字，或者内联元素被设置了 white-space: nowrap;

CSS尺寸分为内部尺寸和外部尺寸

外部尺寸

格式化宽度

格式化宽度仅仅出现在 绝对定位模型 中，也就是出现在 position: absolute/fixed的元素中。
当然在默认情况下，绝对定位元素的宽度的表现应该是包裹性，即宽度由内部尺寸决定。
但是对于 非替代元素，当 left/right 或 top/bottom 对立方的属性值同时存在时，元素的宽度表现为 格式化宽度， 其宽度大小相对于最近的具有定位特性（position的属性值不是static）的祖先元素计算。
格式化宽度也具有完全的流体特性，也就是 margin/border/padding 和 content 内容区域自动分配水平（和垂直）空间。

内部尺寸

表示宽度由内部元素决定

包裹性（包裹+自适应）

所谓 自适应性，指的是元素尺寸由内部元素元素决定，但永远小于 包含块 容器的尺寸（除非！容器尺寸比元素的首选最小宽度还小）
换言之，包裹性元素有个max-width: 100%;来罩着。

类似的 <button> 也是有上述效果的，它是典型的 inline-block元素
容器的文字越多宽度越宽（内部尺寸特性），但是如果文字足够多，则会在容器的宽度处自动换行
作用：
需求： 页面某个模块的文字内容是动态的，设计师经常希望我们做处理，字数少的时候居中显示，文字超过一行的时候居左显示。

最大宽度

指的是元素可以有的最大宽度
“最大宽度”实际等于”包裹性”元素设置了 white-space: nowrap; 之后的宽度。
具体计算：如果内部没有块级元素，或者块级元素没有设定宽度值，则”最大宽度”实际上是最大的连续内联盒子的宽度。
最常见的作用不用js的横向scroll

width值的作用

之前说过内在盒子负责宽高，内容呈现，所以可以说width是作用在 内在盒子 上的。
内在盒子的组成（盒模型）：margin（透明）+ border + padding + content
内在盒子又被分成了4个盒子，分别是 content box 、 padding box 、 border-box 、 margin box。
这四个盒子在CSS中也有对应的名字， content-box 、 padding-box 、 border-box ，就是没有 margin-box ，
因为 margin的背景永远是透明的 ，margin设定具体的值，也不会影响本身的尺寸。因为它是透明的，看不见，没有感觉。因此作为 box-sizing 的属性值就没有了意义，然后就被抛弃了。
padding-box 之前火狐支持过
CSS2.1规定，content box 环绕着width 和 height给定的矩形。
换言之，就是说content box的宽高就是这个元素设置CSS的width、height的值。
box-sizing 的默认值是 content-box 也就好记了。
给元素设置了具体的width之后，会使流动性丢失，需要我们自己去计算宽度排版。

height: 100%;

对于width属性，就算父元素的width为auto, 其百分比也是支持的。但是，对于height属性，如果父元素的height为auto，只要子元素还在文档流中，其百分比值是没有用的。

为什么 height: 100%; 是无效的？
因为CSS规范中，说如果包含块的高度没有显示指定（即高度由内容决定），并且该元素不是绝对定位，则计算值为auto。
auto 百分比 = NAN
而宽度，规则中没有明确说是auto，需要各个浏览器自己去理解发挥（未定义行为）。不过所有的浏览器的理解表现恰巧是一样的。
因为宽度设置百分比，会拿真实的计算值 百分比。

如何使height: 100%有效

  1. 给父元素设置显示的高度值，height: 600px;
  2. 使用绝对定位

min-width/max-width和min-height/max-height

min-width/min-height的默认值是auto
max-width/max-height的默认值是none
